# 网络缓存管理系统 — 接口规范文档

**版本**：v1.0  
**日期**：2026-07-02  
**作者**：第二组  

---

## 1. 概述

`CacheServerClient` 是界面层与通信层之间的**核心接口契约**。所有缓存操作均通过此接口完成，界面层不依赖任何具体实现。

---

## 2. 接口定义

### `CacheServerClient`

**包路径**：`com.cache.client.net.CacheServerClient`

### 2.1 连接管理

#### `void connect(String host, int port)`

| 项目 | 说明 |
|------|------|
| 描述 | 连接到缓存服务器 |
| 参数 | `host` — 服务器主机名/IP；`port` — 服务器端口 |
| 返回 | 无 |
| 异常 | 连接失败时由实现类自行抛出运行时异常 |
| Mock行为 | 仅设置 connected = true，不做实际网络连接 |

#### `void disconnect()`

| 项目 | 说明 |
|------|------|
| 描述 | 断开与缓存服务器的连接 |
| 参数 | 无 |
| 返回 | 无 |
| Mock行为 | 设置 connected = false |

#### `boolean isConnected()`

| 项目 | 说明 |
|------|------|
| 描述 | 检查当前连接状态 |
| 参数 | 无 |
| 返回 | `true` — 已连接；`false` — 未连接 |

### 2.2 缓存数据操作

#### `Optional<String> get(String key)`

| 项目 | 说明 |
|------|------|
| 描述 | 根据 key 获取缓存值 |
| 参数 | `key` — 缓存键，不能为 null 或空 |
| 返回 | 存在且未过期返回 `Optional.of(value)`，否则返回 `Optional.empty()` |
| 注意 | 如果 key 已过期，实现类应主动删除并返回空 |

#### `boolean set(String key, String value, long ttlSeconds)`

| 项目 | 说明 |
|------|------|
| 描述 | 设置一个缓存条目（key 已存在则覆盖） |
| 参数 | `key` — 缓存键；`value` — 缓存值；`ttlSeconds` — 过期时间（秒），<= 0 表示永不过期 |
| 返回 | `true` — 成功；`false` — 失败 |

#### `boolean delete(String key)`

| 项目 | 说明 |
|------|------|
| 描述 | 删除指定 key 的缓存条目 |
| 参数 | `key` — 要删除的缓存键 |
| 返回 | `true` — 删除成功（或 key 不存在）；`false` — 删除失败 |

#### `boolean clear()`

| 项目 | 说明 |
|------|------|
| 描述 | 清空所有缓存条目 |
| 参数 | 无 |
| 返回 | `true` — 清空成功 |

### 2.3 查询与统计

#### `List<String> keys(String pattern)`

| 项目 | 说明 |
|------|------|
| 描述 | 列出匹配指定模式的所有 key |
| 参数 | `pattern` — 通配符模式，支持 `*`（匹配任意多个字符）和 `?`（匹配单个字符） |
| 返回 | 匹配的 key 列表，无匹配时返回空列表 |
| 示例 | `keys("user:*")` → `["user:1001", "user:1002"]` |

#### `Map<String, CacheEntry> getAll()`

| 项目 | 说明 |
|------|------|
| 描述 | 获取所有缓存条目（不含已过期条目） |
| 参数 | 无 |
| 返回 | key → CacheEntry 的不可变映射 |

#### `Map<String, String> stats()`

| 项目 | 说明 |
|------|------|
| 描述 | 获取服务器统计数据 |
| 参数 | 无 |
| 返回 | 统计信息的键值映射，至少包含以下字段： |

**stats() 返回字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `entries` | String | 当前缓存条目数 |
| `memory` | String | 占用内存（如 "1.2 MB"） |
| `hit_rate` | String | 缓存命中率（如 "87.3%"） |
| `connections` | String | 当前连接数 |
| `uptime` | String | 服务器运行时间 |

## 3. 数据模型

### `CacheEntry`

**包路径**：`com.cache.client.model.CacheEntry`

| 字段 | 类型 | 说明 |
|------|------|------|
| `key` | String | 缓存键 |
| `value` | String | 缓存值 |
| `ttlSeconds` | long | 过期时间（秒），<= 0 表示永不过期 |
| `createTime` | Instant | 条目创建时间 |

### 方法

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `isExpired()` | boolean | 如果 `ttlSeconds > 0` 且当前时间已超过 `createTime + ttlSeconds` 则返回 true |

## 4. 实现类规范

### 4.1 MockCacheClient

| 项目 | 说明 |
|------|------|
| 用途 | 开发调试阶段使用，不依赖服务端 |
| 数据存储 | `ConcurrentHashMap<String, CacheEntry>` 内存存储 |
| 预置数据 | 启动时填充 5 条示例数据（user:1001, user:1002, session:abc123, config:theme, rate:limit） |
| 线程安全 | 是（使用 ConcurrentHashMap） |
| 过期处理 | get() 时检查过期并清除 |

| 连接方式 | Socket，由 `connect(host, port)` 建立 |
| 协议格式 | 待第一组确认后定稿（见概要设计 6.2 节草案） |
| 重连机制 | 待实现 |

## 5. 使用示例

```java
// 使用 Mock 模式开发
CacheServerClient client = new MockCacheClient();
client.connect("localhost", 6379);

// 设置缓存
client.set("username", "Alice", 300);

// 获取缓存
Optional<String> value = client.get("username");
value.ifPresent(System.out::println);  // 输出: Alice

// 通配符查询
List<String> keys = client.keys("user:*");

// 统计信息
Map<String, String> stats = client.stats();

// 删除
client.delete("username");

// 清空
client.clear();

// 断开连接
client.disconnect();
```
