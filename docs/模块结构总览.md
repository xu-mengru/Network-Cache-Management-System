## 三、各模块详细职责

---

### 3.1 入口模块 — `CacheClientApp`

| 属性 | 说明 |
|------|------|
| **文件** | `CacheClientApp.java` |
| **类型** | JavaFX Application |
| **依赖** | Config, MockCacheClient / RespCacheClient |

**核心职责**：
1. 应用启动时调用 `Config.load()` 读取配置文件
2. 根据 `config.properties` 的 `client.mock` 选项决定客户端实现：
   - `true` → 创建 `MockCacheClient`（开发、演示用）
   - `false` → 创建 `RespCacheClient`（联调、部署用）
3. 将创建好的 Client 实例作为全局静态变量持有
4. 加载 FXML、创建窗口并显示

**外部接口**：
- `CacheClientApp.getConfig()` → 全局配置对象
- `CacheClientApp.getClient()` → 全局客户端实例

**通信方式**：静态方法访问（全局单例模式）

---

### 3.2 业务接口层 — `CacheServerClient`

| 属性 | 说明 |
|------|------|
| **文件** | `net/CacheServerClient.java` |
| **类型** | Java Interface |
| **方法数** | 9 个（见下方总表） |

**核心职责**：
- 定义客户端与缓存服务端之间的操作契约
- UI 层（MainController）只依赖此接口，不依赖具体实现
- 支持 Mock / RESP 双模式切换

**接口方法总表**：

```
┌──────────────┬──────────────┬──────────┬─────────────────────────┐
│  类别         │ 方法名       │ 返回类型  │ 说明                    │
├──────────────┼──────────────┼──────────┼─────────────────────────┤
│ 连接管理      │ connect()    │ void     │ 连接到服务端             │
│              │ disconnect() │ void     │ 断开连接                │
│              │ isConnected()│ boolean  │ 连接状态检查             │
├──────────────┼──────────────┼──────────┼─────────────────────────┤
│ Server       │ ping()       │ String   │ 连通性测试（返回 PONG）  │
├──────────────┼──────────────┼──────────┼─────────────────────────┤
│ String 操作  │ get()        │ Optional │ 获取值                  │
│              │ set()        │ boolean  │ 设置值（可选 EX）       │
├──────────────┼──────────────┼──────────┼─────────────────────────┤
│ Key 操作     │ del()        │ int      │ 删除键（变参，返回数量） │
├──────────────┼──────────────┼──────────┼─────────────────────────┤
│ List 操作    │ lpush()      │ int      │ 头部插入（返回长度）     │
│              │ rpush()      │ int      │ 尾部插入（返回长度）     │
│              │ lpop()       │ String   │ 头部弹出（null 表空）   │
│              │ lrange()     │ List     │ 范围获取（支持负索引）   │
├──────────────┼──────────────┼──────────┼─────────────────────────┤
│ TTL          │ ttl()        │ long     │ 查询剩余时间（格式待确认）│
└──────────────┴──────────────┴──────────┴─────────────────────────┘

等待第一组确认：
  SCAN(pattern)  ← 格式待确认
  TTL 返回码约定  ← -1/-2 是否与标准 Redis 一致
```

**通信方式**：
- 对上层（UI）：Java 接口方法调用（同步阻塞）
- 对下层：由具体实现决定（Mock 为堆内调用，Resp 为 TCP 网络调用）

### 3.3 网络通信层 — 具体实现

#### 3.3.1 `MockCacheClient`

| 属性 | 说明 |
|------|------|
| **文件** | `net/MockCacheClient.java` |
| **存储** | `ConcurrentHashMap<String, CacheEntry>` + `ConcurrentHashMap<String, List<String>>` |
| **连接** | 本地开关（不涉及网络） |

**核心职责**：
1. 在内存中模拟服务端全部 9 个命令的行为
2. 维护双存储结构：
   - `stringStore`：STRING 类型条目（带过期检查）
   - `listStore`：LIST 类型条目（CopyOnWriteArrayList 保证并发安全）
   - `allKeys`：全局 key 索引（用于 SCAN 模拟）
3. 提供 `getAllLocalEntries()` 方法（不在接口上），供 UI 本地渲染
4. 预置 5 条 String 数据和 2 条 List 数据作为演示

**不依赖外部**：所有操作在 JVM 堆内完成，零外部依赖。

---

#### 3.3.2 `RespCacheClient`

| 属性 | 说明 |
|------|------|
| **文件** | `net/RespCacheClient.java` |
| **通信** | TCP Socket（java.net.Socket） |
| **协议** | RESP 文本协议 |
| **编解码** | 委托 RespCodec 处理 |

**核心职责**：
1. 通过 TCP Socket 连接到第一组的 C 语言服务端（127.0.0.1:6379）
2. 将每个接口方法调用转换为 RESP 格式的请求并发送
3. 读取服务端返回的 RESP 响应并解析为 Java 对象
4. 管理 Socket 生命周期（连接/断开/超时/异常处理）

**通信方式**：

```
MainController.call()
    → RespCacheClient.method()
        → execute("CMD", args...)
            → RespCodec.encode("CMD", args...)       // Java String → RESP byte[]
            → Socket.OutputStream.write(bytes)       // 发送到网络
            → Socket.InputStream.read(bytes)          // 从网络读取
            → RespCodec.decode(inputStream)           // RESP byte[] → RespResponse
            → 根据响应类型提取返回值
    → 返回给 MainController
```

**外部依赖**：第一组 C 服务端运行在 127.0.0.1:6379

### 3.4 协议编解码模块 — `protocol/` 子包

#### 3.4.1 `RespCodec`

| 属性 | 说明 |
|------|------|
| **文件** | `net/protocol/RespCodec.java` |
| **类型** | 静态工具类（无状态，纯函数） |
| **方法** | `encode()` / `decode()` |

**核心职责**：
1. **编码**：将 Java 字符串数组编码为 RESP 文本格式字节流
   - 输入：`("SET", "key", "value")`
   - 输出：`*3\r\n$3\r\nSET\r\n$3\r\nkey\r\n$5\r\nvalue\r\n`
2. **解码**：从 InputStream 读取并解析 RESP 响应
   - 支持 5 种 RESP 类型：`+` 简单字符串、`-` 错误、`:` 整数、`$` 批量字符串、`*` 数组
   - 递归解析数组中的嵌套元素

**协议格式速查**：

```
客户端发送（RESP 数组）:
  *<参数个数>\r\n
  $<第1参数长度>\r\n<第1参数>\r\n
  $<第2参数长度>\r\n<第2参数>\r\n
  ...

服务端响应:
  +OK\r\n         → 成功简单字符串
  -ERR msg\r\n    → 错误
  :100\r\n        → 整数
  $5\r\nhello\r\n → 批量字符串
  $-1\r\n         → 空值
  *3\r\n...\r\n   → 数组
```

---

#### 3.4.2 `RespResponse`

| 属性 | 说明 |
|------|------|
| **文件** | `net/protocol/RespResponse.java` |
| **类型** | 值对象（不可变） |
| **枚举** | Type: SIMPLE_STRING / ERROR / INTEGER / BULK_STRING / NULL / ARRAY |

**核心职责**：
1. 统一封装 5 种 RESP 响应类型
2. 提供类型安全的访问方法：
   - `asString()` → 获取字符串值（SIMPLE_STRING / ERROR / BULK_STRING）
   - `asInteger()` → 获取整数值（INTEGER）
   - `asArray()` → 获取数组元素（ARRAY）
   - `isNull()` / `isError()` / `getError()` → 状态判断
3. 提供工厂方法：`simpleString()`, `error()`, `integer()`, `bulkString()`, `nullBulk()`, `array()`

### 3.5 界面控制层 — `ui/` 包

#### 3.5.1 `MainController`

| 属性 | 说明 |
|------|------|
| **文件** | `ui/MainController.java` |
| **依赖** | `CacheServerClient`（接口）, `CacheEntry` |
| **数据** | `ObservableList<CacheEntry> tableData`（TableView 数据源） |

**核心职责**：
1. **连接管理**：连接/断开按钮 → 调用 client.connect/disconnect
2. **PING 测试**：PING 按钮 → 调用 client.ping() → 显示结果
3. **CRUD 操作**：Add/Delete/ClearAll → 调用 client.set/del → 刷新表格
4. **List 操作**：LPUSH/RPUSH/LPOP/LRANGE 输入 + 按钮 + 结果显示
5. **TTL 查询**：输入 key → 调用 client.ttl() → 显示剩余时间
6. **数据展示**：TableView 绑定 ObservableList，展示全部条目
7. **本地搜索**：搜索框 → 对本地 tableData 过滤（不依赖 KEYS 命令）
8. **数据导出**：JSON/CSV → 委托 ExportUtil
9. **状态栏**：显示条目数量和当前模式（Mock / RESP）

**FXML 控件结构**：

```
MainView (BorderPane)
├── Top: 连接工具栏 (HBox)

│   ├── CRUD 区域: keyField / valueField / ttlField / addBtn / deleteBtn / clearBtn
│   ├── List 操作区域: listKeyField / listValueField / lpushBtn / rpushBtn / lpopBtn / lrangeBtn
│   │   └── listResultView (ListView) / listLengthLabel
│   └── TTL 查询区域: ttlKeyField / ttlQueryBtn / ttlResultLabel
├── Center: 数据表格 (TableView)
│   ├── 列: key / type / value / ttlSeconds / createTime / status
│   └── 底部: searchField / searchBtn / showAllBtn / exportJsonBtn / exportCsvBtn
└── Bottom: 状态栏 (Label)
```

## 四、模块间通信方式总结

| 发送方 | 接收方 | 通信方式 | 协议/格式 |
|--------|--------|---------|----------|
| MainController | CacheServerClient | Java 接口调用（同步） | 方法参数 + 返回值 |
| RespCacheClient | 第一组 C 服务端 | TCP Socket | RESP 文本协议 |
| RespCacheClient | RespCodec | 静态方法调用 | Java String[] / byte[] |
| RespCodec | RespResponse | 工厂方法创建 | Java 值对象 |
| MainController | CacheEntry | 直接引用 | Java POJO 字段访问 |
| MainController | ExportUtil | 静态方法调用 | Collection<CacheEntry> → 文件 |
| CacheClientApp | Config | 静态方法调用 | properties → Config 对象 |
| CacheClientApp | CacheServerClient | 工厂创建（config 驱动） | new Mock/RespClient |

## 五、两种运行模式对比

| 维度 | Mock 模式 | RESP 模式 |
|:----:|----------|----------|
| 配置 | client.mock=true | client.mock=false |
| 客户端 | MockCacheClient | RespCacheClient |
| 存储 | JVM ConcurrentHashMap | 第一组 C 服务端内存 |
| 网络 | 无 | TCP 127.0.0.1:6379 |
| 协议 | 堆内方法调用 | RESP 文本协议 |
| 启动依赖 | 无 | 需服务端运行 |
| 调试 | 断点直接调试 | 需起服务 + telnet 辅助 |
| 演示 | 离线独立演示 | 需演示环境运行服务端 |
| 数据持久性 | 关 JVM 即丢失 | 服务端不关就不丢 |
