# 网络缓存管理系统 — 小组分工方案

**版本**：v1.0  
**日期**：2026-07-02  

---

## 各成员职责总表

|    角色   |            开发阶段任务              | 文档/PPT 任务 |
|-----------|------------------------------------|--------------|
| **组长**  | 架构维护、协议草案、代码合并、进度把控 | 概要设计最终定稿、答辩 PPT 主线、协调文档整合 |
| **组员A** |         缓存表格 + CRUD 操作界面     | 详细设计（界面部分）、功能演示场景设计 |
| **组员B** | 连接管理 + 统计面板 + **后期转 TcpCacheClient** | 详细设计（通信层部分）、协议设计说明 |
| **组员C** | 搜索过滤 + 批量操作 + 数据导出 + **单元测试** | 测试用例设计、测试报告、实习日志 |

---

## 组长

### 开发期
- 维护 `CacheServerClient` 接口和项目骨架
- 制定协议草案（与第一组协商）
- 审核组员的 Pull Request 并合并
- 最后阶段集成 TcpCacheClient 联调

### 文档
- 概要设计（整体架构、模块划分、技术栈）
- 答辩 PPT（串讲全组内容，整合各组员素材）


---

## 组员A

### 开发期
- 缓存条目表格（展示、排序、分页）
- 添加/修改/删除功能（弹窗表单）
- TTL 状态显示（过期高亮、剩余时间倒计时）

### 文档
- 详细设计中「界面模块」部分（界面截图 + 操作流程说明）


---

## 组员B

### 开发期（前期）
- 连接对话框（服务器地址/端口输入）
- 连接/断开按钮、连接状态指示灯
- 统计信息面板（条目数、命中率、内存占用等）

### 开发期（协议确定后）
- 转向实现 `TcpCacheClient`（Socket 连接、协议编解码、重连机制）

### 文档
- 详细设计中「通信层」部分（协议格式、连接流程图）
- 协议设计说明文档


---

## 组员C

### 开发期
- 搜索过滤条（按 key 模式实时搜索）
- 批量删除 / 一键清空
- 数据导出（JSON / CSV 格式）
- 为全组代码编写单元测试

### 文档
- 测试用例设计文档（功能测试 + 边界场景 + 异常场景）
- 测试报告（测试结果截图 + 覆盖率数据）
- 实习日志（每天记录遇到的问题、分析过程和解决方式）
