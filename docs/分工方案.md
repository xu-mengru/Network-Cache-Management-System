## 新增工作说明

### 1. TLV 编解码模块（组员B）
第一组采用 TLV 协议通信，格式为 `Length(4B) + Type(1B) + ArgCnt(1B) + Args(变长)`。
`TcpCacheClient` 需要将 `CacheServerClient` 接口的每个方法调用编码为 TLV 请求，
并将服务端返回的 TLV 响应解码为 Java 对象。

**对应文件**：`src/main/java/com/cache/client/net/TcpCacheClient.java`（骨架已创建，含 TODO）

### 2. 多客户端面板（组员B）
允许在一个窗口内同时运行多个客户端实例，每个实例独立连接服务端、独立展示缓存数据。
类似于浏览器的多标签页，每个标签页就是一个独立的客户端会话。
演示时可以在一个窗口内开多个标签页，分别连接不同服务端或同一服务端，直观展示多客户端并发能力。

**对应文件**：`src/main/java/com/cache/client/ui/MainController.java`（连接管理区域已标注 TODO）

### 3. 新增接口方法（组长定义，各组员实现）
- `boolean exists(String key)` — 判断 key 是否存在
- `boolean expire(String key, long seconds)` — 设置 key 的过期时间
- `long ttl(String key)` — 查询 key 剩余生存时间
- `String type(String key)` — 查询 key 的数据类型

**对应文件**：
- 接口定义：`CacheServerClient.java`（组长已完成）
- Mock 实现：`MockCacheClient.java`（组员A: exists/expire/ttl，组员C: type）
- UI 操作：`MainController.java`（组员A: onCheckKey/onSetExpire）
- 单元测试：`MockCacheClientTest.java`（组员C: 5个测试方法骨架）
- TCP 实现：`TcpCacheClient.java`（组员B: 待联调时实现）
