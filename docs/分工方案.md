# 网络缓存管理系统 — 小组分工方案（最终版）

**版本**：v3.0  
**日期**：2026-07-06  
**协议变更说明**：第一组协议已从 TLV（二进制）变更为 RESP（文本协议），本项目框架已完成对应适配。

## 组员A

### 开发期
- **CRUD 界面重构**：`onAdd()` 改为弹出对话框、`onDelete()` 增加确认对话框、`onClearAll()` 确认后逐个 DEL
- **List 操作 UI 面板**：新增 LPUSH/RPUSH/LPOP/LRANGE 四个操作的输入框、按钮、结果显示（ListView + 长度标签）
- **statusColumn 增强**：自定义 cellFactory 显示数据类型（STRING / LIST）和状态
- **导出适配**：修改 `ExportUtil` 适配 CacheEntry 新增的 type 和 listLength 字段
- 表格排序支持（点击列头排序）

### 文档
- 详细设计中「界面模块」部分（界面截图 + 操作流程说明）
- 功能演示场景设计

## 组员B

### 开发期（核心变更 — 原 TLV 改为 RESP）
- **`RespCodec` 实现**：RESP 编码（命令 → `*n\r\n$len\r\ndata\r\n...`）和解码（输入流 → `RespResponse`），覆盖 + - : $ * 五种类型
- **`RespResponse` 封装**：RESP 响应类型的 Java 对象包装，提供 `asString()/asInteger()/asArray()` 等便捷访问
- **`RespCacheClient` 实现**：基于 Socket + `RespCodec` 的 RESP-TCP 真实客户端，实现接口所有方法
- 连接管理 + 连接状态指示 + 超时/重连配置
- **PING 按钮**：连通性测试
- **TTL 查询面板**：输入 key 显示剩余秒数
- **多客户端标签页**（架构保留，时间充裕时实现）：一个窗口内多个标签页独立连接

### 文档
- 详细设计中「通信层」部分（RESP 协议格式、连接流程图）
- RESP 协议设计说明文档（对比 TLV 的变更说明）
