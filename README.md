# Network Cache Management System

网络缓存管理客户端程序 — 实习课题第二组。

## 项目背景

本项目为西安电子科技大学实习课题，实现**网络缓存管理程序设计**（题目 2），与题目 1（基于 Linux 系统的网络缓存结构设计）配合完成：

- **题目 1** — Linux 服务端：实现缓存核心逻辑（Hash/链表/队列存储、过期策略、并发访问），类似 redis
- **题目 2（本组）** — 客户端：提供可视化图形界面，通过自定义协议连接并管理服务端缓存

## 功能模块

| 模块       | 说明                                       |
|------------|--------------------------------------------|
| 连接管理   | 配置服务端 IP/端口，连接/断开，连接状态显示 |
| 键值浏览   | 以表格形式展示缓存中的 key-value，支持搜索与过滤 |
| 增删改操作 | 添加、修改、删除缓存条目                   |
| TTL 管理   | 查看/设置每条缓存的过期时间                |
| 实时监控   | 缓存命中率、内存用量、连接数等统计信息展示 |
| 批量操作   | 批量删除、清空缓存、按模式匹配 key         |

## 技术栈

- **语言**: Java
- **UI 框架**: JavaFX + FXML + Scene Builder
- **网络**: Java Socket / NIO
- **构建**: Maven / Gradle

## 架构设计

```
┌──────────────────────────────┐    自定义 TCP 协议    ┌──────────────────────┐
│   JavaFX 客户端 (本组)       │ ◄──────────────────► │  Linux 缓存服务端     │
│                              │     (协议待定)        │   （题目 1 组）      │
│  ┌────────────────────────┐  │                      │                      │
│  │  FXML 界面层           │  │                      │  Hash 存储           │
│  │  (Controller)          │  │                      │  过期策略            │
│  ├────────────────────────┤  │                      │  epoll 并发          │
│  │  CacheServerClient     │  │                      │                      │
│  │  （接口层）             │  │                      │                      │
│  ├────────────────────────┤  │                      │                      │
│  │  MockClient /          │  │                      │                      │
│  │  TcpClient             │  │                      │                      │
│  └────────────────────────┘  │                      │                      │
└──────────────────────────────┘                      └──────────────────────┘
```

- **CacheServerClient**（接口层）：定义与服务端交互的操作契约，界面层不直接依赖具体实现
- **MockCacheClient**：返回模拟数据，用于界面层独立开发调试
- **TcpCacheClient**：真实通信实现（待协议确定后完成）

## 开发计划

### Day 1–5：基础准备
- 搭建 JavaFX 开发环境（JDK、JavaFX SDK、Scene Builder）
- 小组学习 JavaFX 核心组件（TableView、FXML、事件处理、Socket 编程）
- 完成界面原型设计（手绘 / Figma）
- 与服务端组初步沟通，了解协议方向

### Day 5–7：界面骨架与 Mock 层
- 搭建主窗口、Tab 页布局、连接对话框等静态界面
- 定义 `CacheServerClient` 接口及数据模型（CacheEntry 等）
- 实现 `MockCacheClient`，界面层基于模拟数据先行跑通
- 准备项目开发文档初稿

### Day 8：文档答辩
- 提交项目开发文档（概要设计、详细设计等）
- 演示当前开发进度

### Day 9–13：核心功能实现与联调
- 根据确定后的通信协议实现 `TcpCacheClient`
- 与服务端组联调
- 完成键值 CRUD、TTL 管理、搜索过滤、实时监控等功能
- 异常处理（断线重连、超时、错误提示）

### Day 14–15：完善与最终交付
- 测试用例与测试脚本
- UI/UX 优化
- 撰写测试报告、答辩 PPT
- 提交全部程序源码与文档
