## 三、RESP协议设计

### 3.1 为什么选 RESP

| 维度 | RESP 文本协议 | TLV 二进制协议 |
|------|:-----------:|:------------:|
| 开发成本 | **低**（~50行解析） | 高（~150行解析） |
| 调试难度 | **易**（telnet可测） | 难（需专用工具） |
| 工具支持 | **丰富**（redis-cli） | 无 |
| 带宽效率 | 中 | 高 |
| 解析步骤 | **5步** | 8步 |

**结论**：9天开发周期，选 RESP 更适合快速交付。

### 3.2 RESP 五种数据类型

| 类型 | 首字节 | 格式 | 示例 |
|------|:------:|------|------|
| 简单字符串 | `+` | `+<内容>\r\n` | `+OK\r\n` |
| 错误 | `-` | `-<错误信息>\r\n` | `-ERR unknown cmd\r\n` |
| 整数 | `:` | `:<数字>\r\n` | `:3\r\n` |
| 批量字符串 | `$` | `$<长度>\r\n<数据>\r\n` | `$5\r\nhello\r\n` |
| 空值 | `$` | `$-1\r\n` | 表示 NULL |
| 数组 | `*` | `*<元素数>\r\n<各元素>` | `*2\r\n$3\r\nGET\r\n$3\r\nkey\r\n` |

### 3.3 请求格式（客户端 → 服务器）

**客户端总是发送 RESP 数组**：

```
SET name Alice

编码为：
  *3\r\n          ← 3个元素
  $3\r\n          ← 第1个参数长度3
  SET\r\n         ← "SET"
  $4\r\n          ← 第2个参数长度4
  name\r\n        ← "name"
  $5\r\n          ← 第3个参数长度5
  Alice\r\n       ← "Alice"
```

### 3.4 响应格式（服务器 → 客户端）

| 命令 | 请求 | 响应 |
|------|------|------|
| SET | `*3\r\n$3\r\nSET\r\n...` | `+OK\r\n` |
| GET（有值） | `*2\r\n$3\r\nGET\r\n...` | `$5\r\nAlice\r\n` |
| GET（无值） | `*2\r\n$3\r\nGET\r\n...` | `$-1\r\n` |
| LPUSH | `*3\r\n$5\r\nLPUSH\r\n...` | `:3\r\n` |
| LRANGE | `*4\r\n$6\r\nLRANGE\r\n...` | `*3\r\n$1\r\na\r\n$1\r\nb\r\n$1\r\nc\r\n` |
| PING | `*1\r\n$4\r\nPING\r\n` | `+PONG\r\n` |

### 3.5 解析实现

```c
// protocol.c - RESP 解析器（约50行）

// 处理流程: *<argc>\r\n$<len1>\r\n<data1>\r\n$<len2>\r\n<data2>\r\n...

int parse_resp(Client *c) {
    char *p = c->read_buf;
    char *nl;
    
    // 1. 解析数组头 *<argc>\r\n
    if (p[0] != '*') return PROTO_ERR;
    nl = strstr(p, "\r\n");
    c->argc = atoi(p + 1);
    
    // 2. 逐个解析 $<len>\r\n<data>\r\n
    p = nl + 2;
    for (int i = 0; i < c->argc; i++) {
        if (p[0] != '$') return PROTO_ERR;
        nl = strstr(p, "\r\n");
        int len = atoi(p + 1);
        p = nl + 2;
        c->argv[i] = strndup(p, len);
        p += len + 2;
    }
    return PROTO_OK;
}

// 编码辅助函数
void reply_string(Client *c, const char *s);
void reply_bulk(Client *c, const char *s, int len);
void reply_int(Client *c, long long n);
void reply_error(Client *c, const char *err);
void reply_null(Client *c);
void reply_array_start(Client *c, int count);
```

## 四、并发模型设计

### 4.1 epoll ET 模式

```c
// event.c - 事件循环

void event_loop(Server *srv) {
    struct epoll_event events[MAX_EVENTS];
    
    while (srv->running) {
        // 等待事件，超时100ms（给定时任务执行机会）
        int n = epoll_wait(srv->epfd, events, MAX_EVENTS, 100);
        
        for (int i = 0; i < n; i++) {
            int fd = events[i].data.fd;
            
            if (fd == srv->listen_fd) {
                // 新连接事件
                accept_clients(srv);
            }
            else if (events[i].events & EPOLLIN) {
                // 可读事件：接收客户端请求
                handle_read(srv, get_client_by_fd(srv, fd));
            }
            else if (events[i].events & EPOLLOUT) {
                // 可写事件：发送响应
                handle_write(srv, get_client_by_fd(srv, fd));
            }
        }
        
        // 每 100ms 定时任务：过期键清理
        evict_expired_keys(srv->db);
    }
}
```

### 4.2 关键技术点

| 要点 | 实现 |
|------|------|
| 触发模式 | **ET（边缘触发）**——只通知一次，必须循环读/写直到 EAGAIN |
| 非阻塞 | `fcntl(fd, F_SETFL, O_NONBLOCK)` |
| 缓冲区 | 每个 Client 独立读写缓冲区 |
| 单线程 | 无锁竞争，无上下文切换，简化开发 |

### 4.3 accept 处理（ET 模式）

```c
void accept_clients(Server *srv) {
    while (1) {
        int client_fd = accept(srv->listen_fd, NULL, NULL);
        if (client_fd == -1) {
            if (errno == EAGAIN || errno == EWOULDBLOCK)
                break;  // 所有连接已accept完毕
            return;
        }
        // 设为非阻塞 + 注册到 epoll
        set_nonblocking(client_fd);
        add_epoll_event(srv->epfd, client_fd, EPOLLIN | EPOLLET);
        
        // 创建 Client 结构
        Client *c = create_client(client_fd);
        srv->clients[client_fd] = c;
    }
}
```

## 五、数据结构设计

### 5.1 Hash 表（渐进式 Rehash）

```c
// ============ 哈希节点 ============
typedef struct DictEntry {
    void *key;
    void *value;
    struct DictEntry *next;   // 链地址法解决冲突
    uint64_t hash;
} DictEntry;

// ============ 哈希表 ============
typedef struct DictHashTable {
    DictEntry **table;        // 桶数组
    unsigned long size;       // 桶数量（2的幂）
    unsigned long sizemask;   // size - 1，用于位运算取模
    unsigned long used;       // 已用桶数
} DictHashTable;

// ============ 字典（支持渐进式Rehash） ============
typedef struct Dict {
    DictHashTable ht[2];      // 双表
    long rehash_idx;          // -1 表示未在rehash，>=0 表示当前迁移的桶索引
} Dict;
```

#### 扩容/缩容触发条件

| 操作 | 触发条件 | 新大小 |
|------|----------|--------|
| **扩容** | `used / size > 1.0`（负载因子 > 1） | `size × 2` |
| **缩容** | `used / size < 0.1`（负载因子 < 0.1） | `size / 2`（最小4） |

#### 渐进式迁移流程

```c
// 每次读写操作前执行一步迁移（迁移1个桶，约10个节点）
void dict_rehash_step(Dict *d) {
    if (d->rehash_idx == -1) return;
    
    // 跳过空桶
    while (d->rehash_idx < d->ht[0].size 
           && d->ht[0].table[d->rehash_idx] == NULL)
        d->rehash_idx++;
    
    // 检查是否完成
    if (d->rehash_idx == d->ht[0].size) {
        free(d->ht[0].table);
        d->ht[0] = d->ht[1];
        memset(&d->ht[1], 0, sizeof(DictHashTable));
        d->rehash_idx = -1;
        return;
    }
    
    // 迁移当前桶的所有节点
    DictEntry *entry = d->ht[0].table[d->rehash_idx];
    while (entry) {
        DictEntry *next = entry->next;
        unsigned long idx = entry->hash & d->ht[1].sizemask;
        entry->next = d->ht[1].table[idx];
        d->ht[1].table[idx] = entry;
        d->ht[1].used++;
        d->ht[0].used--;
        entry = next;
    }
    d->ht[0].table[d->rehash_idx] = NULL;
    d->rehash_idx++;
}

// 查找：查两个表
DictEntry* dict_find(Dict *d, const void *key) {
    if (d->rehash_idx != -1) dict_rehash_step(d);
    
    uint64_t h = hash_func(key);
    for (int t = 0; t <= 1; t++) {
        unsigned long idx = h & d->ht[t].sizemask;
        DictEntry *entry = d->ht[t].table[idx];
        while (entry) {
            if (entry->hash == h && key_compare(entry->key, key) == 0)
                return entry;
            entry = entry->next;
        }
        if (d->rehash_idx == -1) break;  // rehash已完成，只查ht[0]
    }
    return NULL;
}

// 写入：rehash期间写入ht[1]
void dict_add(Dict *d, void *key, void *value) {
    if (d->rehash_idx != -1) dict_rehash_step(d);
    int target = d->rehash_idx != -1 ? 1 : 0;
    // ... 插入到 ht[target]
}
```

### 5.4 键过期管理

```c
typedef struct Database {
    Dict *data;      // key → RedisObject*
    Dict *expires;   // key → expire_time (毫秒时间戳)
    int id;          // 数据库编号
} Database;
```

**两种删除策略**：

| 策略 | 触发时机 | 实现 |
|------|----------|------|
| **惰性删除** | 访问 key 时检查 | `db_get()` 中检查是否过期 |
| **定期删除** | 每 100ms 定时 | 随机抽取20个带过期时间的key，删除已过期的 |

```c
// 惰性删除
void* db_get(Database *db, const char *key) {
    int64_t *expire = dict_find(db->expires, key);
    if (expire && *expire < current_time_ms()) {
        // 键已过期
        dict_delete(db->data, key);
        dict_delete(db->expires, key);
        return NULL;
    }
    return dict_find(db->data, key);
}

// 定期删除
void evict_expired_keys(Database *db) {
    for (int i = 0; i < 20; i++) {     // 每次检查20个
        char *key = dict_random_key(db->expires);
        if (!key) break;
        int64_t *expire = dict_find(db->expires, key);
        if (*expire < current_time_ms()) {
            dict_delete(db->data, key);
            dict_delete(db->expires, key);
        }
    }
}
```

### 6.3 命令分发

```c
void dispatch_command(Client *c, int argc, char **argv) {
    // 线性查找命令表（8个命令，性能足够）
    for (int i = 0; cmd_table[i].name != NULL; i++) {
        if (strcasecmp(cmd_table[i].name, argv[0]) == 0) {
            // 参数数量校验
            if (argc < cmd_table[i].min_arity) {
                reply_error(c, "ERR wrong number of arguments");
                return;
            }
            if (cmd_table[i].max_arity != -1 && argc > cmd_table[i].max_arity) {
                reply_error(c, "ERR wrong number of arguments");
                return;
            }
            // 执行命令
            cmd_table[i].func(c, argc, argv);
            return;
        }
    }
    reply_error(c, "ERR unknown command");
}
```

```c
// cmd_string.c —— SET 命令
void cmd_set(Client *c, int argc, char **argv) {
    // 解析可选参数 EX
    long long expire_ms = 0;
    if (argc == 5) {
        if (strcasecmp(argv[3], "EX") == 0) {
            expire_ms = atoll(argv[4]) * 1000;
        }
    }
    
    // 写入数据库
    RedisObject *obj = create_string_object(argv[2]);
    dict_replace(c->server->db->data, argv[1], obj);
    
    // 设置过期时间

        int64_t expire_time = current_time_ms() + expire_ms;
        dict_replace(c->server->db->expires, argv[1], &expire_time);
    }
    
    reply_string(c, "OK");
}

// cmd_list.c —— LRANGE 命令
void cmd_lrange(Client *c, int argc, char **argv) {
    RedisObject *obj = dict_find(c->server->db->data, argv[1]);
    if (!obj || obj->type != OBJ_LIST) {
        reply_null(c);
        return;
    }
    
    List *list = obj->ptr;
    int start = atoi(argv[2]);
    int stop  = atoi(argv[3]);
    
    // 处理负索引
    if (start < 0) start = list->len + start;
    if (stop < 0)  stop  = list->len + stop;
    if (start < 0) start = 0;
    if (stop >= list->len) stop = list->len - 1;
    
    // 计算元素个数
    int count = (start <= stop) ? (stop - start + 1) : 0;
    
    // 编码 RESP 数组响应
    reply_array_start(c, count);
    
    // 定位起始节点
    ListNode *node = list->head;
    for (int i = 0; i < start && node; i++)
        node = node->next;
    
    // 遍历输出
    for (int i = 0; i < count && node; i++) {
        char *val = (char*)node->value;
        reply_bulk_element(c, val, strlen(val));
        node = node->next;
    }
}
```

### 8.2 AOF 实现（可选，~100行代码）

```c
// 写命令记录
void aof_append(Server *srv, int argc, char **argv) {
    if (!srv->aof_enabled) return;
    // 只记录写命令
    static const char *write_cmds[] = {"SET","DEL","LPUSH","RPUSH","LPOP",NULL};
    if (!is_write_command(argv[0], write_cmds)) return;
    
    // 写入文本: SET key value\n
    for (int i = 0; i < argc; i++)
        fprintf(srv->aof_fp, "%s%s", i > 0 ? " " : "", argv[i]);
    fprintf(srv->aof_fp, "\n");
    fflush(srv->aof_fp);
}

// 启动加载（重放命令恢复数据）
void aof_load(Server *srv) {
    FILE *fp = fopen("appendonly.aof", "r");
    if (!fp) return;
    
    char line[1024];
    while (fgets(line, sizeof(line), fp)) {
        int argc; char **argv = parse_line(line, &argc);
        execute_internal(srv, argc, argv);  // 执行但不响应客户端
        free_argv(argc, argv);
    }
    fclose(fp);
}
```

### 9.2 一致性哈希（16384 槽位）

```c
#define SLOT_COUNT 16384  // Redis Cluster 标准

// 槽位计算
int get_slot(const char *key) {
    return crc16(key, strlen(key)) % SLOT_COUNT;
}

// 路由
int get_node(int slot, int node_count) {
    return slot / (SLOT_COUNT / node_count);
}
```

### 9.3 与简单取模分片的对比

| 方案 | 添加节点 | 数据迁移量 |
|------|----------|:----------:|
| **取模分片** `hash(key) % N` | N→N+1 | **~80%** 需迁移 |
| **一致性哈希** 16384 槽位 | N→N+1 | **~1/(N+1)** 需迁移 |

### A. RESP 协议速查

```
客户端 → 服务器：
  *<argc>\r\n$<len1>\r\n<arg1>\r\n$<len2>\r\n<arg2>\r\n...

服务器 → 客户端：
  +OK\r\n                    简单字符串
  -ERR message\r\n           错误
  :100\r\n                   整数
  $5\r\nhello\r\n            批量字符串
  $-1\r\n                    空值
  *3\r\n...\r\n              数组
```

```
PING [msg]         → +PONG\r\n 或 +msg\r\n
SET k v [EX sec]   → +OK\r\n

DEL k1 [k2 ...]    → :n\r\n
LPUSH k v1 [v2..]  → :len\r\n
RPUSH k v1 [v2..]  → :len\r\n
LPOP k             → $len\r\nv\r\n 或 $-1\r\n
LRANGE k start end → *n\r\n...
```
